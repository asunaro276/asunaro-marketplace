# プロジェクト規約テンプレート

**注意:** このファイルは、プラグイン導入先で実際のプロジェクト規約を作成するためのテンプレートです。以下の内容を参考に、プロジェクト固有の規約を記述してください。

---

## このファイルの使い方

プラグインを導入したプロジェクトで、このファイルを以下のように更新してください：

1. プロジェクトで使用している命名規則を記述
2. CSS/スタイリングの規約を定義
3. Vue コンポーネントの構造規則を明記
4. プロジェクト固有のベストプラクティスを追加

---

## 記述すべき内容の例

### 1. ファイル命名規則
- Vue コンポーネントのファイル名形式
- CSS/SCSS ファイルの命名ルール

### 2. CSS クラス命名規則
- 使用する命名規則（BEM, SMACSS, など）
- プロジェクト固有のプレフィックスやネームスペース

### 3. デザイントークン
- CSS 変数の配置場所
- 変数の命名パターン

### 4. Vue コンポーネント構造
- SFC（単一ファイルコンポーネント）の構造
- Props の定義方法
- スタイルのスコープ（scoped / module）

### 5. アクセシビリティ要件
- セマンティック HTML の使用
- ARIA 属性の規約

### 6. レスポンシブデザイン
- ブレークポイントの定義
- メディアクエリの記述方法

### 7. 禁止事項
- インラインスタイル
- グローバルスコープの汚染
- など

---

**重要:** この規約ファイルは、Figma デザイン実装スキルが `references/project-convention.md` を読み込んで従うため、プロジェクト開始時に必ず作成してください。
